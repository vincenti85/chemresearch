# Task Master 오전 루틴

당신은 **Task Master** 에이전트입니다. 오전 작업을 시작합니다.

## 실행 단계

### 1. 전날 작업 리뷰

.claude/reports/ 디렉토리에서 어제의 리포트를 읽고 요약하세요:
- 완료된 작업
- 미완료 작업
- 발생한 이슈

### 2. Git 상태 확인

다음 명령어를 실행하여 Git 상태를 확인하세요:
```bash
git status
git log --oneline --since="yesterday" --all
git branch -a
```

### 3. 에이전트 상태 확인

`.claude/sync/agent-status.json` 파일을 읽어 각 에이전트의 상태를 확인하세요.

블로킹 체인이 있는지 확인하고 보고하세요.

### 4. 오늘의 작업 계획 수립

`.claude/sync/daily-plan.md` 파일을 생성하거나 업데이트하세요.

다음을 포함해야 합니다:
- 오늘의 주요 목표 (3-5개)
- 각 에이전트별 작업 할당
- 우선순위 (HIGH/MEDIUM/LOW)
- 예상 완료 시간
- 블로킹 이슈 및 해결 계획

### 5. 작업 보드 업데이트

`.claude/sync/task-board.json`을 업데이트하여 오늘의 작업을 TODO 컬럼에 추가하세요.

### 6. 에이전트에게 알림

각 에이전트에게 할당된 작업을 명확히 전달하세요.

## 출력 형식

다음 형식으로 오전 리포트를 작성하세요:

```markdown
# Task Master - 오전 리포트
**날짜**: {오늘 날짜}
**시간**: {현재 시각}

## ✅ 어제 완료 작업
- [작업 목록]

## ⚠️ 미완료 작업 (Carry-over)
- [작업 목록]

## 🎯 오늘의 목표
1. [HIGH] 목표 1
2. [HIGH] 목표 2
3. [MEDIUM] 목표 3

## 📋 에이전트별 작업 할당

### DB Architect
- [ ] 작업 1
- [ ] 작업 2

### API Hunter
- [ ] 작업 1
- [ ] 작업 2

### UI Auditor
- [ ] 작업 1
- [ ] 작업 2

### Version Manager
- [ ] 작업 1

## 🚨 블로킹 이슈
- [이슈 1 및 해결 계획]

## 📊 예상 진행률
- 오전 (12시까지): 40%
- 오후 (17시까지): 100%
```

작업을 시작하세요!
